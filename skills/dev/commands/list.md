# /dev:list — 개발 문서 목록 조회

## ★ 최우선: .dev/DEV.md 가이드라인 확인

**이 명령을 실행하기 전에 반드시 `.dev/DEV.md` 파일이 존재하는지 확인한다.**

- `.dev/DEV.md`가 **존재하면**: 먼저 이 파일을 읽고, 그 안에 기록된 가이드라인에 따라 동작한다. DEV.md의 지시사항은 이 스킬 문서의 기본 동작보다 **우선**한다.
- `.dev/DEV.md`가 **존재하지 않으면**: 이 스킬 문서의 기본 동작 그대로 진행한다.

DEV.md는 프로젝트별 커스텀 가이드라인이다. 예를 들어 list 출력 형식이나 추가 표시 항목 등의 지시가 있을 수 있다.

---

## 1. 이 명령의 목적

`.dev/` 폴더에 있는 모든 기능/프로젝트의 개발 문서 현황을 한눈에 보여준다.

ARGUMENTS 없이 바로 실행 가능하다.

---

## 2. 실행 방법

### 2.1 .dev/ 폴더 스캔

`.dev/` 폴더의 모든 하위 디렉터리를 스캔한다.

각 폴더에서 확인할 파일:
- `README.md` — copy 참조 문서 (있으면 copy 완료)
- `plan.md` — 구현 명세서 (있으면 plan 완료)
- `progress.md` — 실행 진행 상태 (있으면 execute 진행/완료)

### 2.2 상태 판별 기준

| 상태 | 조건 | 아이콘 |
|---|---|---|
| **참조만** | README.md만 존재 (copy만 된 상태) | `[copy]` |
| **계획 수립** | plan.md 존재, 상태가 Draft/Reviewed | `[plan:Draft]` 또는 `[plan:Reviewed]` |
| **실행 대기** | plan.md 존재, 상태가 Ready, progress.md 없음 | `[plan:Ready]` |
| **실행 중** | progress.md 존재, 완료되지 않은 Phase 있음 | `[executing]` |
| **실행 완료** | plan.md 상태가 Completed | `[done]` |

### 2.3 출력 형식

다음 형식으로 사용자에게 보여준다:

```
## .dev/ 개발 문서 현황

| # | 이름 | 상태 | 명세 점수 | Phase 진행 | 설명 |
|---|------|------|----------|-----------|------|
| 1 | 게시판 | [plan:Ready] | 97/100 | - | 게시글 CRUD + 댓글 + 좋아요 |
| 2 | 채팅 | [executing] | 95/100 | 2/4 완료 | Flutter 실시간 채팅 |
| 3 | 기존인증 | [copy] | - | - | 현재 세션 기반 인증 참조 |
| 4 | 인증리팩토링 | [done] | 98/100 | 3/3 완료 | JWT 전환 완료 |

총 4개 항목 | copy: 1 | plan: 1 | executing: 1 | done: 1
```

---

## 3. 세부 정보 표시

각 항목의 정보를 추출하는 방법:

### 이름
- `.dev/` 아래 디렉터리명

### 상태
- 위 2.2 기준으로 판별

### 명세 점수
- `plan.md`가 있으면 "명세 완전성 점수" 줄에서 추출
- 없으면 `-`

### Phase 진행
- `progress.md`가 있으면 완료/전체 Phase 수 추출
- `plan.md`만 있으면 전체 Phase 수만 표시
- 없으면 `-`

### 설명
- `plan.md`의 Intent에서 추출
- `plan.md`가 없으면 `README.md`의 개요에서 추출
- 한 줄로 축약

---

## 4. .dev/ 폴더가 없는 경우

`.dev/` 폴더가 존재하지 않으면:

```
.dev/ 폴더가 아직 없습니다.

시작하려면:
  /dev:copy <대상> <이름> — 기존 코드를 참조 문서로 복사
  /dev:plan <이름> — 새 기능의 구현 명세서 작성
```

---

## 5. .dev/ 폴더가 비어있는 경우

`.dev/` 폴더는 있지만 하위 디렉터리가 없으면:

```
.dev/ 폴더는 있지만 개발 문서가 없습니다.

시작하려면:
  /dev:copy <대상> <이름> — 기존 코드를 참조 문서로 복사
  /dev:plan <이름> — 새 기능의 구현 명세서 작성
```
